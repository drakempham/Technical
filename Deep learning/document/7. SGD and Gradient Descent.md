# 7. SGD and Gradient Descent

## Các thông số cơ bản

### Model: Linear Regression
```
ŷ = w*x + b
```

| Thông số | Tên | Ý nghĩa |
|---|---|---|
| `w` | weight | Độ dốc — x tăng 1 thì ŷ tăng bao nhiêu |
| `b` | bias | Điểm xuất phát khi x = 0 |
| `lr` | learning rate | Mỗi bước update đi xa bao nhiêu |
| `L` | loss | Đo mức độ sai của dự đoán |

### Loss function: Mean Squared Error
```
L = (ŷ - y)²
```
Bình phương để: âm thành dương, sai nhiều bị phạt nặng hơn.

---

## 4 bước của 1 iteration SGD

### Bước 1: Forward pass
```
ŷ = w*x + b
```
Model dùng w, b hiện tại để **đoán** kết quả.

### Bước 2: Tính Loss
```
L = (ŷ - y)²
```
**Đo sai số** giữa dự đoán và thực tế.

### Bước 3: Backward pass — tính Gradient (Chain Rule)
```
∂L/∂ŷ = 2(ŷ - y)

∂ŷ/∂w = x          ∂ŷ/∂b = 1

→ ∂L/∂w = 2(ŷ - y) * x      ← có x vì w gắn với x trong công thức
→ ∂L/∂b = 2(ŷ - y) * 1      ← không có x vì b đứng một mình
```

Gradient = **hướng mà nếu tăng tham số thì Loss tăng**.
Muốn giảm Loss → đi **ngược chiều** gradient.

### Bước 4: Update weights
```
w = w - lr * ∂L/∂w
b = b - lr * ∂L/∂b
```

---

## Ví dụ tính tay: 1 sample

**Setup:** x=1, y=2, w=0, b=0, lr=0.1

```
Forward:  ŷ = 0*1 + 0 = 0
Loss:     L = (0 - 2)² = 4

Gradient:
  ∂L/∂w = 2(0-2)*1 = -4      ← âm → tăng w thì Loss giảm
  ∂L/∂b = 2(0-2)   = -4

Update:
  w = 0.0 - 0.1*(-4) = 0.4
  b = 0.0 - 0.1*(-4) = 0.4

Kiểm tra: ŷ = 0.4*1 + 0.4 = 0.8   (gần y=2 hơn rồi)
```

## SGD với batch size = 1, 5 samples

**5 samples:** (1,2), (2,4), (3,5), (4,4), (5,5)  
**Khởi tạo:** w=0, b=0, lr=0.1

| Sample (x,y) | ŷ | Loss | ∂L/∂w | ∂L/∂b | w sau | b sau |
|---|---|---|---|---|---|---|
| (1, 2) | 0.0 | 4.00 | -4.0 | -4.0 | 0.40 | 0.40 |
| (2, 4) | 1.2 | 7.84 | -11.2 | -5.6 | 1.52 | 0.96 |
| (3, 5) | 5.52 | 0.27 | 3.12 | 1.04 | 1.208 | 0.856 |
| (4, 4) | 5.69 | 2.85 | 13.5 | 3.38 | -0.142 | 0.518 |
| (5, 5) | -0.19 | 26.95 | -51.9 | -10.4 | 5.05 | 1.56 |

**Nhận xét:** Loss dao động mạnh (4 → 26.95) vì mỗi sample kéo w theo hướng khác nhau — đặc trưng của SGD batch=1.

## Epoch là gì

**1 epoch** = đi qua tất cả n samples **1 lần**.

```
Khởi tạo w, b
      │
      ▼
┌─────────────────┐
│  Epoch 1        │
│  Sample 1 → update
│  Sample 2 → update
│  ...            │
│  Sample n → update
└────────┬────────┘
         │ w, b mới
         ▼
┌─────────────────┐
│  Epoch 2        │  ← lặp lại cùng n samples với w, b mới
│  ...            │
└────────┬────────┘
         ▼
    Loss ≈ 0 → DỪNG
```

**Total loss qua từng epoch (ví dụ 5 samples trên):**
```
Epoch 1: L_total = 41.9
Epoch 2: L_total ≈ 35.2
Epoch 3: L_total ≈ 18.4
...
Epoch 50: L_total ≈ 0.8   ← gần hội tụ
```

## Điều kiện dừng

| Điều kiện | Ví dụ |
|---|---|
| Hết số epoch định trước | `epochs = 100` |
| Loss đủ nhỏ | `L < 0.001` |
| Loss không giảm thêm | validation loss tăng lại → **early stopping** |

---

## SGD vs Batch GD vs Mini-batch GD

| | SGD | Mini-batch GD | Batch GD |
|---|---|---|---|
| Batch size | 1 | 32 / 64 / 128 | toàn bộ dataset |
| Update sau mỗi | 1 sample | k samples | n samples |
| Tốc độ update | Nhanh | Trung bình | Chậm |
| Noise | Cao (dao động nhiều) | Vừa | Thấp |
| Memory | Ít nhất | Vừa | Nhiều nhất |
| Dùng thực tế | Ít | **Phổ biến nhất** | Ít (dataset lớn) |

---

## Learning rate ảnh hưởng thế nào

```
lr quá lớn (lr=10):    w nhảy quá xa, dao động không hội tụ
                        Loss: 4 → 100 → 50 → 200 → ...

lr quá nhỏ (lr=0.0001): đi quá chậm
                        Loss: 4 → 3.99 → 3.98 → ... (cần hàng triệu epoch)

lr vừa (lr=0.1):       hội tụ ổn định
                        Loss: 4 → 2 → 0.5 → 0.1 → ...
```

---

## Hình dung gradient descent

```
     Loss
      │  .
      │ . .
      │.   .
      │     .← đang đứng đây
      │      .
      └──────────── w
            ↑
       muốn đến đây (Loss thấp nhất)

Gradient = hướng lên dốc
Update    = bước 1 bước nhỏ xuống dốc (ngược gradient)
```

---

## Code Python đầy đủ

```python
# 5 samples
data = [(1,2), (2,4), (3,5), (4,4), (5,5)]
w, b, lr = 0.0, 0.0, 0.1

for epoch in range(50):
    total_loss = 0
    for x, y in data:
        # Forward
        y_hat = w * x + b
        # Loss
        loss = (y_hat - y) ** 2
        total_loss += loss
        # Gradient
        dL_dw = 2 * (y_hat - y) * x
        dL_db = 2 * (y_hat - y)
        # Update
        w -= lr * dL_dw
        b -= lr * dL_db
    
    if epoch % 10 == 0:
        print(f"Epoch {epoch}: Loss={total_loss:.3f}, w={w:.3f}, b={b:.3f}")
```
